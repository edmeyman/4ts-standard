## Why Dual Licensing?

### Specification (CC BY-NC-ND 4.0)

The specification text is protected to ensure:

1. **Standard Integrity:** No fragmentation through unauthorized modifications
2. **Quality Control:** All conformance claims reference the canonical standard
3. **Attribution:** Clear provenance and stewardship
4. **Non-commercial Access:** Free for research, education, and evaluation

### Schemas & Code (MIT)

The schemas and tools are permissively licensed to enable:

1. **Open Tooling:** Any organization can use the published schemas, vectors, and reference code
2. **Commercial Use of the Code:** The published code and schemas carry no copyright licensing fees
3. **Modification:** Implementers can adapt the published reference code to their needs
4. **Broad Adoption:** No copyright barriers to referencing and testing against 5TS

The MIT license governs copyright in the listed files. It does not grant patent rights; see the Patent Rights section below.

## Conformance Claims

Organizations may publicly claim conformance to the v1.0.2 conformance bundle without permission if they:

1. Pass all conformance tests (8/8 vectors)
2. Publish conformance claim with format:
   ```
   Tool@Version • PCD-1 • Bundle-1.0.2 • 8/8 • manifest_hash • logs_url
   ```
3. Make verification logs publicly available
4. Do not imply endorsement beyond technical conformance

False conformance claims may result in revocation of permission to use 5TS trademarks.
